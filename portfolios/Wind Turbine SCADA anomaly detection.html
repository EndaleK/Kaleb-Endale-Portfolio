Executive Summary

This report presents a comprehensive analysis of the Wind Turbine SCADA Anomaly Detection project, which aims to enable early detection of critical failures in wind turbine operations. The system analyzes 95 multi-year time-series events from 36 turbines across three geographically distributed wind farms, leveraging unsupervised machine learning to identify anomalous operational patterns that precede failures.

Key Insight #1: Hydraulic Failures Dominate the Failure Landscape

Finding: Hydraulic group failures account for 50% of all anomalies in Wind Farm A, representing the single largest failure mode. This pattern reveals hydraulic systems as the primary vulnerability in wind turbine operations.

Critical Statistics:
  • Hydraulic group: 6 events (50%)
  • Gearbox failure: 3 events (25%)
  • Generator bearing failure: 2 events (17%)
  • Transformer failure: 1 event (8%)

Implications

  • Prioritized Monitoring: Deploy enhanced monitoring on hydraulic oil temperature (sensor_41)
  • Early Warning Window: Hydraulic failures show gradual temperature increases 24-72 hours before failure
  • Maintenance Strategy: Implement predictive maintenance schedules focused on hydraulic system health
  • Cost Impact: While less catastrophic than gearbox failures, hydraulic issues cause frequent downtime

Operational Recommendations

  • Set hydraulic oil temperature thresholds at ±2σ from normal operating range
  • Implement automated alerts when temperature trends exceed 5°C increase over 24 hours
  • Schedule preventive hydraulic system inspections every 2000 operating hours
  • Maintain hydraulic fluid quality through quarterly contamination analysis

Key Insight #2: Extreme Feature Heterogeneity Across Wind Farms

Finding: Wind farms exhibit an 11x variation in feature counts (86 to 957 sensors), creating unprecedented data heterogeneity. This massive disparity reflects different turbine generations, manufacturers, and sensor instrumentation philosophies.

Wind Farm Features Turbines Location

Technical Challenges

Memory Management: Wind Farm C with 957 features × 50,000+ rows ≈ 48M data points requires 12-16GB RAM per dataset. Solution: Process farms separately with chunked loading.

Architectural Solutions

  • Farm-Specific Pipelines: Modular processing approach for each farm
  • Feature Selection: Reduce to common feature subset (top 100) before modeling
  • Computational Requirements: Minimum 32GB RAM for Farm C processing
  • Transfer Learning: Models trained on Farm A won't transfer to Farm C

Key Insight #3: 10-Minute Sampling Creates 1-2 Day Detection Window

Finding: The 10-minute sampling interval combined with train/test split structure provides a 1-2 day early warning window for most fault types. This temporal resolution strikes an optimal balance between data volume and actionable lead time.

Key Insight #4: Temperature Sensors Are Primary Failure Predictors

Finding: Temperature sensors demonstrate the highest discriminative power for anomaly detection, with bearing and gearbox temperature features showing Cohen's d effect sizes of 0.8-1.5 (large to very large effects).

Critical Temperature Monitoring Points

Sensor Description Cohen's d Effect Size
sensor_11_avg Gearbox bearing temp 1.45 Very Large
sensor_13_avg Generator bearing 2 1.32 Very Large
sensor_12_avg Gearbox oil temp 1.18 Large
sensor_14_avg Generator bearing 1 1.12 Large
sensor_41_avg Hydraulic oil temp 0.95 Large
Feature Engineering Impact: Best engineered feature (rolling std 6h) outperforms raw temperature by 48% (d=2.15 vs d=1.45)

Physical Interpretation

  • Friction Increases: Bearing degradation → increased friction → heat generation
  • Lubrication Breakdown: Oil quality degradation → poor lubrication → elevated temps
  • Electrical Resistance: Winding insulation damage → increased resistance → heat
  • Fluid Dynamics: Coolant flow restriction → heat accumulation → temperature spikes

Methodology Overview

Phase 1: Project Setup

Created Python virtual environment, configured YAML-based configuration management, established directory structure

Phase 2: Data Exploration

Loaded 95 events from 3 wind farms, validated schema consistency, profiled feature distributions

Phase 3: Data Cleaning

Removed duplicates, handled missing values, detected and capped outliers, normalized angle features

Phase 4: Exploratory Data Analysis

Generated distributions, calculated Cohen's d effect sizes, identified top 30 discriminative features

Phase 5: Feature Engineering

Created rolling statistics, lag features, temperature gradients, power curve features (200-400 new features per dataset)

Phase 6: Anomaly Detection Modeling

Trained 4 models on 51 normal events, evaluated on all 95 events, created weighted ensemble

Phase 7: Visualization & Dashboard

Built interactive Streamlit multi-page dashboard with real-time anomaly detection interface
